# 向量时钟分布式日志聚合系统 — Linux/UDP 版

> 基于向量时钟因果排序 + 倒排索引的分布式日志采集与聚合系统，C 语言实现。

## 项目简介

本项目是一个分布式日志聚合系统的教学实现，核心功能包括：

- **向量时钟（Vector Clock）**：实现分布式事件的因果排序，解决消息乱序到达问题
- **倒排索引（Inverted Index）**：支持中文 bigram + 英文空格分词的全文检索
- **因果排序缓冲区（Causal Buffer）**：根据向量时钟拓扑排序，保证日志按因果顺序交付
- **UDP 网络传输**：多 Agent 节点通过 UDP 将日志发送至聚合 Server
- **文件监控**：Linux inotify 事件驱动，实时捕获日志文件变更
- **持久化**：日志以 JSON Lines 格式落盘，重启可恢复

## 快速开始

### 编译

```bash
cd linux_udp
make          # 编译全部
make clean    # 清理
```

**依赖**：gcc、pthread、Linux inotify（仅 agent 需要）

### 方式一：单进程演示（最快体验）

无需网络和文件，直接观察因果排序 + 搜索效果：

```bash
make demo
# 或
./demo_all
```

### 方式二：集成测试

```bash
make test
# 或
./test_e2e
```

### 方式三：多进程完整运行

打开 3~4 个终端：

```bash
# 终端 1：启动服务器
./server

# 终端 2：节点 A 的 Agent
./agent A logs_A.txt

# 终端 3：日志生成器，模拟 A 产生日志
./log_gen logs_A.txt

# 终端 4（可选）：节点 B 的 Agent
./agent B logs_B.txt
```

## 查询命令

在 server 终端中输入：

```
search 转账               # 全文搜索（AND 匹配）
search 转账 --node A      # 限定节点
search 超时 --from 2026-07-01 --to 2026-07-06   # 时间范围
search --or 超时 失败      # OR 匹配
list                      # 列出所有日志
list A                    # 列出节点 A 的日志
count                     # 统计总数
errors                    # 错误次数
exit                      # 退出
```

## 技术细节

### 向量时钟

- 每个节点维护一个分布式计数器映射
- 本地事件：对应节点计数器 +1
- 收到消息：取逐分量最大值，自身计数器 +1
- 支持 4 种关系判断：before / after / equal / concurrent

### 因果排序缓冲区

- 新消息进入缓冲区后，检查向量时钟依赖
- 只有所有前置依赖都已交付的消息才会被弹出
- 支持循环扫描拓扑排序，批量交付有序消息

### 倒排索引

- 中文文本使用 **bigram** 分词
- 英文文本按 **空格** 分词
- 支持 AND 搜索、OR 搜索、时间范围过滤、节点过滤

### 文件监控

- Agent 使用 Linux **inotify** 事件驱动监听
- 文件被删除/轮转时自动重置偏移量
- 非轮询模式，低 CPU 占用

## 运行参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| UDP 端口 | 服务器监听端口 | 9876 |
| 服务器地址 | Agent 连接的 IP | 127.0.0.1 |
| 持久化文件 | JSON Lines 格式 | `logs.jsonl` |

自定义地址和端口：

```bash
./agent A logs_A.txt 192.168.1.100 9876
./server 8888
```
